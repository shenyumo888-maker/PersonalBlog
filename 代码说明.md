

🧱 一、项目总体结构（3个 app）

整个项目目录大概是这样：

PythonClassProject/
│
├── manage.py
├── db.sqlite3
├── myproject/             ← 主配置目录（settings、urls、wsgi）
│
├── blog/                  ← 博客系统（核心功能）
├── accounts/              ← 用户注册、登录
└── comments/              ← 评论功能


⸻

📰 二、核心功能说明

我给你的那份代码是一个完整可运行的入门级博客网站框架，功能如下👇

（1）博客模块（blog app）

主要文件：models.py, views.py, urls.py, templates/blog/

功能：
	•	首页：显示所有博文标题与内容摘要
	•	详情页：点击标题可查看完整内容
	•	后台：可以用管理员账号在后台添加、修改、删除文章
	•	模型 Post 包含字段：

title = models.CharField(max_length=100)
body = models.TextField()
created = models.DateTimeField(auto_now_add=True)
updated = models.DateTimeField(auto_now=True)



也就是说，它已经能实现一个最简化的“博客系统”：
	•	用户访问 / → 看到所有文章列表
	•	管理员在 /admin → 创建文章 → 前端自动显示

⸻

（2）用户模块（accounts app）

功能：
	•	用户注册、登录、注销
	•	登录后可以看到一个简单的“个人中心”页面 /accounts/profile/
	•	支持管理员后台管理用户

这部分基于 Django 自带的认证系统扩展，非常轻量但实用。

⸻

（3）评论模块（comments app）

功能：
	•	在博文详情页下方可以添加评论
	•	评论与 Post 一对多关联（一个文章多个评论）
	•	管理员后台可以审核或删除评论

⸻

（4）静态与模板系统

目录：

templates/
    base.html              ← 全站模板框架
    blog/post_list.html    ← 博客首页
    blog/post_detail.html  ← 博客详情页
    accounts/login.html    ← 登录页
    comments/form.html     ← 评论表单

每个页面都继承自 base.html，有一个统一的导航栏（首页、登录、退出等）。

⸻

⚙️ 三、项目运行流程

运行后你可以：
	1.	启动服务：

python manage.py runserver


	2.	打开浏览器访问：
	•	http://127.0.0.1:8000/ → 博客首页
	•	http://127.0.0.1:8000/admin/ → 管理后台
	•	http://127.0.0.1:8000/accounts/login/ → 登录
	3.	登录后台 → 创建博文 → 首页显示


🌟 五、总结一句话

这套代码是一个“轻量级可运行的博客网站”，包含文章系统、用户系统、评论系统，并且你可以在此基础上继续扩展（比如点赞、标签、分页、富文本编辑器等）。
